# Обучение от тети Аннушки

Публичный учебный сайт с модулями по программированию, файловыми материалами и скрытой системой комментариев без пользовательской авторизации.

## Что это за проект

Сайт построен как библиотека учебных модулей:

- каждый модуль это отдельная страница-урок
- у модуля есть текстовые секции, примеры и файлы
- все страницы открываются публично, без входа
- у каждого модуля есть скрытый блок комментариев
- пользователь может отправлять вопрос, файлы и скриншоты
- оператор отвечает на отдельной внутренней странице `/operator`

## Как работает комментарийный сценарий

### Пользователь

- открывает модуль
- открывает скрытый блок комментариев
- пишет проблему
- может приложить несколько файлов
- может вставить скриншот прямо из буфера
- отправляет комментарий
- видит статус `Подождите... решаем проблему`
- получает ответ оператора без обновления страницы

### Оператор

- открывает `/operator`
- видит входящие комментарии автоматически
- скачивает файлы пользователя
- пишет ответ
- может приложить несколько файлов к ответу
- сохраняет ответ
- ответ сразу появляется у пользователя

## Переменные окружения

### Обязательные для локального dev

- `DATABASE_URL` — строка подключения к PostgreSQL
- `NEXT_PUBLIC_SITE_URL` — базовый URL приложения

### Для storage

- `NEXT_PUBLIC_CDN_BASE_URL`
- `S3_ENDPOINT`
- `S3_REGION`
- `S3_BUCKET`
- `S3_ACCESS_KEY`
- `S3_SECRET_KEY`

Сейчас проект может работать локально и без реального CDN. Для dev используются локальные файлы и `public/...`.

### Для антиспама

- `TURNSTILE_SECRET_KEY`
- `NEXT_PUBLIC_TURNSTILE_SITE_KEY`

Если ключи пустые, в development используется локальный bypass.

## Типовые проблемы

### `docker: command not found`

Не установлен Docker Desktop или он не добавлен в `PATH`.

### `failed to connect to the docker API`

Docker установлен, но не запущен. Нужно запустить Docker Desktop.

### `npm: command not found`

Не установлен Node.js или не добавлен в `PATH`.

### `EPERM: operation not permitted, open .next\trace`

Обычно это значит, что у вас уже запущен второй `next dev`.

Решение:

1. Остановить текущий `dev` через `Ctrl + C`
2. Если не помогло, закрыть висящие `node`-процессы
3. Запустить снова:

```powershell
npm run dev
```

### Пустой каталог модулей

Скорее всего не был выполнен:

```powershell
npm run seed
```

### Ошибки комментариев

Проверьте:

- что запущен `PostgreSQL`
- что выполнены миграции
- что `DATABASE_URL` корректный
- что размер файла не превышает `20 MB`
- что тип файла входит в разрешенный список
